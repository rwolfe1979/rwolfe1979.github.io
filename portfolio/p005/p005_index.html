P-005 · App Evaluation Framework

Marketplace App Evaluation Framework

Before any Marketplace app went into production, I ran it through the same evaluation: compatibility with our Jira version, performance impact in sandbox, functional testing of the specific features we actually needed, and a cost vs. value check. I also documented a rollback procedure before deploying anything. Most apps didn't make it past sandbox.

The Problem

App requests came in without any structure. A team would find something in the Marketplace, ask for it, and expect it to be live by the end of the week. There was no process for evaluating whether an app was safe, compatible, or actually worth the cost.

A bad Marketplace app in a 300-user environment can cause performance issues or break existing automation rules. I needed a repeatable process that protected the environment while still being responsive to legitimate requests.
